String Interpolation

Angular uses a concept called databinding. This boils down to how we can conenct our front end (View) to our backends. (Components). Any data in our components needs to have a way to be transferred to the view. One way to do this, is to use String Interpolation.

Another data-binding option we have is called 'property binding'. It is only capable of bringing data from the component to the view as well. Not vice versa.

Property binding is a technique where we can make the value of some element property equal to a result as adefined in teh typescript file. String interpolation is sueful for evaluating data form teh backend and displaying it on the fornt end. Property binding is used to ttake values from the backend and apply them as properties for the elements.

EVENT BINDING
Event binding is a form of one-way data binding that goes from the frontend to the components. We take action from the html and have it call a function in the component.

Property binding uses square brackets on an element property

Event binding uses parenthesis around some event.
