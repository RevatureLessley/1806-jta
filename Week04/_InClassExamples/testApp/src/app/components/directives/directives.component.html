DIRECTIVES

Directives are ways that we can invoke functionality within an html file. They act as syntax that can be used in html format without breaking the front end.

NOTE: Rule of thumb is that if it is a non html element/attribute and it preceded with 'ng', chances are you have a directive on your hands.

*ngIf

What does the asterisk(*) entail in front of the ngIf? (Asked Ryan's class unaminously)

Ryan:Any directive that has an asterisk(*) means that it is a structural directive. This means the directive is capable of changing the actual structure of the HTML itself. IE, it can add/remove elements on the page.

*ngFor

*ngFor is a directive that can add elements based off of some count fo items from an array. For each item we cna create a duplicate representation of the tag and children tags that we apply it to. With the data being different thanks to data binding. (Since this si changing the structure of our HTML page, we call this a structural directive. Also hinted by the asterisk (*) (3 lines on top of each other symmetrically))

NOTE: any directive used as an attribute of an element is called an 'Attribute Directive'.
